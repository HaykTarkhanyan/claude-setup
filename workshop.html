VS Code intercepts some keys that Claude Code needs. Fix in keybindings.json:

KeyFixWhy

2. ccstatusline - Know Your Context

Claude degrades well before hitting 100% context. The ~50% rule: if you're past half, start a fresh session.

What you see:

  • Line 1: Context usage (13%), git branch, staged changes
  • Line 2: Session time, session cost%, token throughput, reset timer, weekly usage
  • Bottom: Permission mode indicator

Context hygiene

SituationAction
Switching to unrelated task/clear
Continuing related work/compact (lossy - prefer fresh session)
Quick side question/btw (reuses prompt cache, cheap)
Context past ~50%Start fresh: claude -n "new-task"

3. Project Setup & Self-Improvement

.claude/ directory structure

.claude/
  settings.json          # team settings (committed)
  settings.local.json    # personal overrides (gitignored)
  hooks/                 # lifecycle scripts
  skills/                # reusable workflows
  agents/                # specialized Claude instances
  rules/                 # path-specific instructions

CLAUDE.md

Project-level instructions Claude reads every conversation. Keep under 200 lines. Include what Claude can't guess: build commands, architecture constraints, conventions.

Settings hierarchy

~/.claude/settings.json (global) > .claude/settings.json (project) > .claude/settings.local.json (personal)

Memory

Built-in memory lives at ~/.claude/projects/<project>/memory/ as flat markdown files. Persists across conversations - no need to re-explain preferences. Good enough for most use cases.

For more structure, MemPalace provides a local-first knowledge graph with temporal validity windows:

  • Verbatim storage - no lossy summarization
  • Method of loci hierarchy: Wings > Rooms > Halls > Closets > Drawers
  • 29 MCP tools for read/write, knowledge graph, agent diaries
  • Auto-save hooks before context compression
  • Cross-session, cross-project recall
  • 100% local - zero API calls
pip install mempalace

PostToolUseAuto-run after editsLint, format, upload configs
SessionStartOrientation briefingGit status, open PRs, reminders
PostCompactContext recoveryRe-inject critical rules

Hook config pattern

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{ "type": "command", "command": "bash .claude/hooks/guard-push.sh" }]

6. Agents & Subagents

Agents are .claude/agents/*.md files - specialized Claude instances with focused instructions and their own context window.

The key insight

Main session orchestrates. Subagents do heavy lifting. Main context stays clean - it only holds summaries, not full workloads.

Patterns

  • Implement + review: one agent codes in a worktree, another reviews the diff
  • Parallel exploration: dispatch multiple research agents simultaneously
  • Background: Ctrl+B to send a task to the background
  • Custom agents: place .claude/agents/reviewer.md and Claude auto-delegates matching tasks

7. Plugins & MCPs

Superpowers - structured development

Enforced 5-phase workflow that prevents the "just start coding" failure mode:

  1. Clarify requirements
  2. Design architecture
  3. Plan implementation steps
  4. Code with TDD (red-green-refactor)
  5. Verify with two-stage review

Features: subagent-driven development, git worktree isolation, systematic debugging with root-cause analysis.

/plugin install superpowers@claude-plugins-official

Caveman - token compression

Strips filler while keeping technical accuracy. Research showed brevity constraints improved accuracy by 26 percentage points.

LevelWhat it does
/caveman liteDrops filler words, keeps grammar
/caveman fullFragments, drops articles
/caveman ultraMaximum telegraphic compression

Code review + simplify

/code-review runs 4 parallel agents: CLAUDE.md compliance, bug scan, git-blame context. Findings scored 0-100.

/simplify - post-implementation cleanup that preserves functionality.

Best workflow: code -> /code-review -> /simplify

Playwright

Browser automation via MCP. Screenshots, form filling, E2E testing. Useful for visual QA and web scraping tasks.

9. Workflow Patterns & Tips

Plan-review-code-review

Plan (plan mode) -> Review plan -> Code (auto mode) -> Review (NEW session)

Always review in a fresh session. The coding session has accumulated bias.

The interview trick

"Interview me about the requirements before you start." Claude asks clarifying questions, you write a spec, then start a fresh session for implementation.

Autoresearch pattern

Karpathy's automated experiment loop - works for any task with a measurable metric:

modify -> evaluate -> keep/discard -> repeat

In his case: 83 runs overnight, 15 kept, 11% performance gain. Use Claude Code's headless mode to automate:

claude -p "try a different approach to X, run tests, report results" > run_N.txt
